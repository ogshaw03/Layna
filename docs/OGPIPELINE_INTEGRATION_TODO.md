# OGPipeline 連携（今後対応予定）

> ステータス: **未着手 / 今後対応予定（planned）**
> このドキュメントは将来の実装のための設計メモ（Git上のログ）です。

## 目的

OGPipeline 側に「パブリッシュ機能」を追加し、OGPipeline のショットリスト等から動画ファイルを選択、
コメント等の必要事項を記入して「パブリッシュ」すると、Layna のプロジェクトフォルダ内の該当フォルダに
情報が格納され、Layna のレビューページに反映される、という連携を実現する。

## 全体方針（2段構え）

Layna のプロジェクトデータ本体（`layna.project.json` 等）はブラウザ側（localStorage / IndexedDB）に
あり、外部ツールが直接ブラウザのDBを書き換えることはできない。そのため次の2段階で連携する。

1. **OGPipeline 側（パブリッシュ機能を追加）**
   - ショットリストから動画ファイルを選択、コメント・作業者・バージョン名・ショット/工程を入力
   - Layna のプロジェクトフォルダ内の該当パスへ格納する:
     - 動画本体 → `<プロジェクトフォルダ>/<ショット>/<工程>/xxx.mp4`
       （Layna が理解する `Shot001/Lay` などのショット/工程フォルダ構成に合わせる）
     - メタ情報のサイドカー → 同フォルダに `publish.json`
       （コメント・作業者・日時・ショット/工程・バージョン名を記載）

2. **Layna 側（取り込み＝同期機能を追加）**
   - プロジェクトフォルダを走査し、DB未登録の「動画＋`publish.json`」を検出
   - 対応する工程（レビューノード）に新バージョンとして登録（無ければ自動作成）
   - コメントはアップロードコメント（`kind:'upload'`）として記録
   - → レビューページ・タイムラインに反映（メディアは `p:<projectId>|...` 参照で指すのでコピー不要）

## 想定するサイドカー `publish.json`（案）

```json
{
  "shot": "Shot001",
  "stage": "Lay",
  "version": "v003",
  "file": "Shot001_Lay_v003.mp4",
  "author": "yamada",
  "comment": "sh001 修正しました。ご確認ください。",
  "publishedAt": "2026-07-06T10:00:00+09:00"
}
```

## 制約・注意点

- ブラウザ単体では **完全自動のフォルダ監視は不可**（File System Access API にファイル監視がない）。
  実運用は「パブリッシュ後に Layna 側で同期ボタンを押す／プロジェクトを開いた時に自動同期」の形になる。
- リアルタイムに自動プッシュしたい場合は、サーバー版（Node.js）にパブリッシュ用 API を追加し、
  OGPipeline から HTTP 送信する構成が向く（複数人共有・競合対策も有利）。
- ショット/工程のフォルダ命名規約を統一しておく必要がある（ズレるとマッピング不可）。
- ブラウザ版での取り込みは Chrome / Edge（File System Access API 対応）が前提。

## 実装オプション（優先度順の候補）

1. **フォルダ取り込み（同期）機能を Layna に追加**（最小・推奨）
   - OGPipeline は既定のフォルダ構成＋`publish.json` を書き出すだけ
   - Layna に「プロジェクトフォルダを走査して未登録の動画を各ショット工程に登録」する同期機能を追加
   - 参考: OGREF_Beta にサブフォルダ走査ロジックがあり流用可能
2. **サーバー版経由のパブリッシュ API**（複数人・リアルタイム向け）
3. **OGPipeline 側の書き出しフォーマット確定 → Layna 取り込み**（1と組み合わせ）
